# 🚀 CRM Application – QA Automation Practice Project

A full-stack **Customer Management System (CRM)** built for **QA Automation testing practice**.  
This project simulates real-world enterprise workflows used in software testing environments.

---

## 📌 Project Objective

This application is designed to help QA Engineers practice:

- UI Automation Testing (Selenium / Playwright)
- API Testing (Postman / REST Assured / Pytest)
- Manual Test Case Design
- End-to-End Testing of real business workflows

---

## 🏗️ Tech Stack

### 🔹 Frontend
- Angular
- HTML, CSS, TypeScript
- Angular Forms & Validation

### 🔹 Backend
- Python (Flask / FastAPI)
- REST APIs
- JSON data handling

### 🔹 Tools (For QA Practice)
- Selenium WebDriver
- Postman
- Pytest / TestNG (optional)
- Git & GitHub

---

## ✨ Key Features

### 🔐 Authentication
- Login functionality
- Form validation (mandatory fields)
- Error handling for invalid credentials

### 🧑‍💼 Customer Management
- Create new customer
- Edit customer details
- Delete customer
- View customer list

### 📋 Validation Rules
- Mandatory field validation
- Email format validation
- Required field checks
- Error messages for invalid inputs
